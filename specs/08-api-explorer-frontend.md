# Feature: API Explorer Frontend

## Goal
Provide a built-in web UI that lists all generated API endpoints, allows interactive testing (send requests, view responses), and displays a real-time performance monitoring dashboard — all without leaving the browser.

## Scope
- In scope: Endpoint listing (REST and GraphQL), request builder (method, path, headers, body), response viewer (status, headers, body), auth token management (login to get token, auto-attach to requests), endpoint search/filter, **real-time performance monitoring dashboard (request rate, latency, DB pool, query stats)**, **pre-filled login credentials from `.env`**, **dark / light / system theme toggle**
- Out of scope: API documentation authoring, collection saving, environment switching, code snippet generation, collaborative features, offline mode, historical data persistence, alerting

## API Contract

### UI Routes (served by FastAPI)

```
GET /explorer          -> API Explorer single-page application
GET /explorer/static/* -> Static assets (JS, CSS)
```

### Backend API used by the frontend

The frontend calls the same APIs that external consumers use:

```
POST /auth/login        -> Get JWT token
GET  /openapi.json      -> Fetch REST endpoint schema (requires auth, auto-generated by FastAPI)
POST /graphql           -> Send GraphQL introspection query + user queries
GET  /metrics           -> Fetch Prometheus-format performance metrics (polled every 5s)
GET  /healthz           -> Fetch service liveness status
GET  /healthz/detail    -> Fetch detailed diagnostics (pool, introspection stats)
GET  /explorer/config   -> Fetch explorer UI config (pre-fill credentials)
```

#### GET /explorer/config

Always public (no auth). Returns JSON:

```json
{
  "username": "admin",
  "password": "secret"
}
```

Both fields are `null` when not configured. The frontend pre-fills the login form with these values (password field is pre-filled but still masked). If both are `null`, the form is empty as before.

Config source: `EXPLORER_USERNAME` and `EXPLORER_PASSWORD` environment variables (or `.env` file).

### Top Navigation Bar
- Two tabs: **API Explorer** | **Dashboard**
- Service status indicator (green dot = healthy, red = unhealthy), polled from `/healthz`
- Current username + logout button (right side)
- **Theme toggle button** (right side, before logout): cycles through `system → light → dark`. Icon reflects current mode (☀ light / ☾ dark / ⊙ system). Choice is persisted in `localStorage` under key `dbzap-theme`.

### Frontend Pages / Views

#### 1. Login Panel
- Username + password form
- On page load, fetch `GET /explorer/config`; if `username` and/or `password` are non-null, pre-fill the corresponding input fields (password is masked)
- Calls `POST /auth/login`, stores `access_token` in `sessionStorage`
- Shows token expiry countdown

#### 2. Endpoint Sidebar
- Lists all REST endpoints grouped by table (extracted from OpenAPI spec)
- Lists `/openapi.json` under a "Schema" group so users can browse the raw OpenAPI spec directly from the explorer (no pagination query params shown)
- Lists all GraphQL queries and mutations (extracted from introspection)
- Search/filter by name
- Click to select an endpoint

#### 3. Request Builder
- Method badge (GET/POST/PUT/PATCH/DELETE) + URL path
- Path parameters: auto-detected input fields (e.g. `{id}`)
- Query parameters: input fields for `page`, `page_size`, etc.
- Request body: JSON editor (for POST/PUT/PATCH/GraphQL)
- Headers editor (pre-filled with `Authorization: Bearer <token>`)
- Response format: auto-generated preview showing the expected response body structure (extracted from OpenAPI schema for REST with `$ref` resolution via `components.schemas`, from GraphQL introspection for GQL)
- "Send" button

#### 4. Response Viewer
- Status code with color indicator (2xx green, 4xx yellow, 5xx red)
- Response headers (collapsible via "Headers" toggle button) inside a card with toolbar row containing section label and copy button
- Response body: formatted JSON inside a card with toolbar row containing section label and copy button
- Copy buttons: SVG clipboard icon + "Copy" label, always visible; hover turns accent-colored; shows "Copied" in green for 1.5s after click
- Response time (ms)

#### 5. Metrics Dashboard

Auto-refreshes every 5 seconds (configurable, pause/resume button).

##### Row 1: Service Status Card
- Uptime duration (from `/healthz`)
- Liveness status (green/red badge)
- Readiness status (green/red badge, from `/healthz/ready`)
- API mode label (`rest` / `graphql` / `both`)
- Introspected table count

##### Row 2: HTTP Overview (from `/metrics` parsing)
- **Request Rate** — requests/sec (calculated from `http_requests_total` delta between polls), rendered as a number with trend arrow (up/down vs previous interval)
- **Active Requests** — current `http_requests_in_progress` gauge, rendered as a number
- **Error Rate** — percentage of 4xx+5xx in last polling window, rendered as a percentage with color (green < 1%, yellow < 5%, red >= 5%)
- **p95 Latency** — estimated from `http_request_duration_seconds` histogram buckets, rendered in ms

##### Row 3: Per-Endpoint Table
- Sortable table: Path | Method | Requests | Errors | p50 | p95 | p99 | Avg
- Data derived from `http_requests_total` and `http_request_duration_seconds` per `{method, path}` label set
- Click a row to highlight that endpoint in the sidebar and switch to API Explorer tab
- Search/filter by path name

##### Row 4: Database Health
- **Connection Pool** — horizontal stacked bar: `checked_out` (blue) + `overflow` (orange) + `available` (green) out of `pool_size`
- **Pool Utilization** — percentage gauge (`checked_out / pool_size`)
- **Query Latency** — p50 / p95 from `db_query_duration_seconds`, grouped by table and operation
- **Top Slow Queries** — top 5 table+operation combos by p95 latency

##### Row 5: Sparklines (last ~60 polls = ~5 minutes)
- Request rate over time (line chart)
- Error rate over time (line chart)
- p95 latency over time (line chart)
- Pool utilization over time (line chart)
- Rendered using `<canvas>` with vanilla JS (no chart library)
- Each sparkline: ~200px wide, 60px tall, minimal axis labels

## Data Model

No new tables. The frontend is a static SPA served by FastAPI.

## Tech Stack

| Layer       | Choice                          | Rationale                                    |
| ----------- | ------------------------------- | -------------------------------------------- |
| Markup      | HTML5                           | Semantic, no build step                      |
| Styling     | CSS3 (vanilla)                  | Lightweight, no framework overhead           |
| Logic       | Vanilla JavaScript (ES modules) | No build toolchain, served as static files   |
| JSON editor | `<textarea>` + format button    | Simple, no heavy dependency                  |
| HTTP client | `fetch()`                       | Native, supports all needed features         |

No Node.js, no npm, no build step. All frontend files live under `src/dbzap/server/static/` and are served by FastAPI's `StaticFiles`.


## Edge Cases
- API Explorer disabled: configurable via `settings.enable_explorer` (default `True`). When `False`, `/explorer` returns 404.
- Not authenticated: show login panel, block request builder until login succeeds.
- Token expired during session: on 401 response, show "Token expired" banner and redirect to login.
- OpenAPI spec unavailable (REST mode disabled): hide REST endpoint section, show only GraphQL if available.
- GraphQL disabled: hide GraphQL section, show only REST endpoints.
- Both disabled (`api_mode` misconfigured): show empty state message.
- Large response body (> 1MB): truncate display with "Show full response" button.
- Network error: show error message with retry button, do not crash the page.
- `$ref` in OpenAPI response schema points to a missing or unknown schema: fall back to showing the raw `$ref` name rather than crashing.
- Circular `$ref` chains (schema A references B which references A): depth limit of 4 prevents infinite recursion, showing `...` at the cutoff.
- Path parameter missing: "Send" button is disabled until all required path params are filled.
- Invalid JSON in request body: highlight error, block send.
- Dashboard polling when tab is hidden (`document.hidden`): pause polling to save resources, resume on visibility change.
- Dashboard polling when `/metrics` is unreachable: show "Metrics unavailable" banner, retry with exponential backoff (5s → 10s → 30s), do not crash.
- Metrics endpoint returns empty body (fresh start, no requests yet): show "No data yet" placeholder on all cards, sparklines render as flat zero lines.
- Sparkline history exceeds 60 data points: drop oldest (FIFO ring buffer), keep chart width constant.
- Per-endpoint table with hundreds of rows: virtual scroll not needed (typically < 50 endpoints), but add search/filter.
- Histogram p95 estimation: use linear interpolation between bucket boundaries; if all requests fall in one bucket, display that bucket's upper bound.
- Rapid polling with slow `/metrics` response: if a poll hasn't returned before the next interval fires, skip the new poll (no request stacking).
- Clicking endpoint row in dashboard switches to Explorer tab: if Explorer is in a different scroll position, scroll to top.
- `EXPLORER_USERNAME` / `EXPLORER_PASSWORD` not set: `/explorer/config` returns `{"username": null, "password": null}`; login form is empty, user must type credentials.
- `/explorer/config` fetch fails (network error): silently ignore, leave login form empty.
- Theme preference `system`: uses `window.matchMedia('(prefers-color-scheme: dark)')` to determine effective theme; updates automatically if system preference changes while page is open.
- Theme preference stored in `localStorage` survives page reload but not across different browser profiles.

## Module Location
- Frontend: `src/dbzap/server/static/`

- Config: `src/dbzap/core/config.py` (add `explorer_username: str | None`, `explorer_password: str | None`)

## Dependencies
- `fastapi.staticfiles.StaticFiles` (serve static assets)
- `src/dbzap/core/config.py` (`enable_explorer`, `explorer_username`, `explorer_password` fields)
- No additional Python or JS dependencies
